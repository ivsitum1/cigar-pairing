# Manuscript Pre-Output Checklist

**Source:** `58_manuscript_agent_protocol.md` Part 5  
**Gate:** Mandatory before delivering any manuscript `.docx` to the user.

---

## 5.1 Pre-output checklist

- [ ] **One file only** — no intermediates, no backups in output folder
- [ ] **Declarations match** — each header corresponds to its content
- [ ] **Abstract numbers match Results** — cross-check every number
- [ ] **Translated abstract** — same section order, same numbers as English
- [ ] **References** — consistent format, sufficient count, DOIs where available
- [ ] **Tables** — no prose-table redundancy; all referenced in text
- [ ] **Figures** — all referenced; legends self-explanatory
- [ ] **Reporting checklist** — STROBE/CONSORT/PRISMA/CARE as applicable; available as supplement or appendix
- [ ] **Word count** — within journal limit
- [ ] **Hedge count** — ≤1 hedge per 300 words in Discussion/Conclusion

---

## 5.2 Weighted self-assessment

Score each dimension 1–10; report weighted average in chat before delivery.

| Dimension | Weight |
|-----------|--------|
| Completeness (required sections) | 20% |
| Statistical rigour | 20% |
| Writing clarity (assertive, non-redundant, precise) | 20% |
| Table/figure quality | 15% |
| Reference adequacy | 10% |
| File hygiene (one file, naming, no artifacts) | 15% |

**Minimum for output: weighted average ≥ 9/10 (canonical per core-principles.mdc).** Iterate if below 9/10. Below 7/10 → re-approach entirely.

---

## 5.3 Common failure modes

1. Header-content permutation in Declarations
2. Backup file proliferation (`_pre_`, datestamps)
3. QA artifacts in output folder
4. Orphaned translated abstract as separate `.md`
5. Progressive degradation (later edits break earlier correct sections)
6. Reference rot (uncited or misrepresented sources)
