# Skill: Presentation and speech

## When to use

- Drafting spoken talks (20–50 min) with stories and clear through-line
- Adapting technical content for general audience
- Delivery notes (pace, pauses, emphasis) when requested

## Procedure

1. **Audience and outcome:** who listens, what they should remember.
2. **Arc:** hook → problem → insight → evidence (1–2 concrete examples) → actionable close.
3. **Script:** short sentences; avoid manuscript jargon unless expert room.
4. **Q&A prep:** 3 likely hard questions with honest answers.
5. **Embodied delivery (optional):** stage movement, pause markers, not over-choreographed.

## Verification

- [ ] One core message, not a literature review
- [ ] Examples concrete; no fabricated anecdotes
- [ ] Time estimate stated
